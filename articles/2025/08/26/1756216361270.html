从零搭建个人博客指南:Solo + GitHub Pages

2025-08-26
   
基础技能
    0 0

本篇指南是为了刚进入大学或者对计算机了解不多，但想要拥有一个属于自己的技术博客的同志们编写的。本篇指南将会用最易懂的方式，带着你一步步构建个人博客，同时厘清个个工具的具体用途，不需要任何编程基础！

为什么要搭建个人博客？

搭建博客除了收获自己的博客之外并不能收获其他东西，所以请思考为什么要搭建个人博客

学会新知识却不记录下来，导致很快就忘了；没有正反馈，持续学习能力弱；你是否也有这种经历？

以下是我搭建个人博客的原因：

  • 📃将自己学习历程（计算机、文学）以及个人心经记录于网上
  • 📜提升写作和表达能力
  • 📄督促自身提升学习技术以及提高项目管理能力

目标：

  • 将每个阶段性的成果发布到个人博客上

整体方案介绍

我们将使用 Solo博客系统 + GitHub Pages 的方式：

  • Solo：一个简单易用的Java博客系统
  • GitHub Pages：GitHub提供的免费静态网站托管服务
  • Git Bash：用于将博客内容上传到GitHub的工具

不用担心自己不会使用Java或者是不会使用Git，唯一需要搜索资料的部分我会在准备工作部分介绍

第四部分：一键提交

初学者可仅仅只学习以上内容，接下来的内容可在熟练掌握Git后进行学习

当我们利用Solo生成页面后，只需要双击一个文件，剩下的所有事情都会自动完成：上传代码、更新网站。整个过程只需要3秒钟，完全不需要手动操作。

自动化脚本组成

  1. deploy.sh :Shell脚本 核心执行文件
  2. deploy.bat:Windows批处理文件 启动器

自动化脚本代码:

  1. deploy.sh：
#!/bin/bash
# 这行告诉系统用bash shell来执行这个脚本

# 文件拷贝 - 从solo博客生成目录拷贝到GitHub页面目录
cp -r /d/blog/solo-v4.4.0/static-site/* /d/blog/zyboshi.github.io/
# cp = copy拷贝命令
# -r = 递归拷贝（包含所有子文件夹和文件）
# 第一个路径是源目录（你的博客生成文件）
# 第二个路径是目标目录（GitHub页面仓库）

echo "开始上传"
# 在终端显示提示信息

# 切换到GitHub页面目录
cd /d/blog/zyboshi.github.io
# cd = change directory切换目录

# 添加所有文件到git暂存区
git add --all
# 将目录中的所有文件标记为待提交状态

echo "请输入提交描述文字如果没有默认: 更新文章"
read describe
# 提示用户输入提交描述
# read命令等待用户输入，并将输入内容存入describe变量

if [ ! -n "$describe" ]
then
describe="更新文章"
fi
# 如果用户没有输入内容（直接按回车），则使用默认描述"更新文章"

echo $describe
# 显示最终使用的提交描述

# 提交代码到本地仓库
git commit -m "$describe"
# commit = 提交更改
# -m = 添加提交消息
# "$describe" = 使用前面设置的描述文字

# 拉取远程代码（防止冲突）
git pull
# 先从GitHub拉取最新代码，确保没有冲突

# 推送代码到GitHub
git push origin main
# push = 推送代码
# origin = 远程仓库别名（通常是GitHub）
# main = 主分支名称（原来是master，现在主流是main）

替换部分:

echo 🚀 开始一键部署博客...

"D:\useful_tool\git\Git\git-bash.exe" -c "cd /d/blog/ && bash deploy.sh"

echo.
echo ✅ 部署完成！按任意键退出...
pause
